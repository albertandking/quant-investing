## 1.1 本章导读：把投资变成可检验的规则

想象两个投资者。

第一个凭感觉：今天看新闻觉得利好，就买点；明天群里有人说要跌，就卖了。他赚了钱说自己有眼光，亏了钱怪市场不理性。一年下来，他说不清自己到底为什么赚、为什么亏，也无法把「经验」复制到明年。

第二个把决策写成了规则：「当一只股票过去 20 天涨幅排进全市场前 10%，且换手率不过高，就买入，持有 10 天再换。」这条规则可以**用历史数据检验**——过去十年这样做的收益、回撤、胜率是多少？可以**复现**——换个人、换台电脑跑出来结果一样；可以**改进**——把 20 天改成 60 天会更好吗？

第二个投资者做的，就是**量化投资**。

本章不写代码细节，而是先建立四个认知，它们决定了你之后学的所有方法是否用得对：

1. 量化投资到底是什么，能做什么、不能做什么；
2. 一套量化策略从想法到落地，要经过哪些环节（本书的地图）；
3. 中国 A 股有哪些**制度特征**（T+1、涨跌停、停牌），它们如何影响策略；
4. 三个最容易让人「回测赚钱、实盘亏钱」的**陷阱**。

最后，我们会把开发环境装好，并跑通全书第一段代码。

## 1.3 什么是量化投资

**量化投资（Quantitative Investing）**，是指**用数学模型和计算机程序，系统化地做出投资决策**的方法。它的对立面不是「价值投资」或「技术分析」——这些投资理念都可以被量化——而是**凭主观感觉、不可复现的随意决策**。

一套典型的量化策略包含三个可被代码表达的部分：

| 部分 | 回答的问题 | 例子 |
|---|---|---|
| **选股 / 择时（信号）** | 买什么、什么时候买 | 买过去一个月动量最强的 50 只股票 |
| **组合构建** | 各买多少、如何分配权重 | 等权、按市值加权、或做风险平价 |
| **风险与执行** | 如何控制回撤、如何下单 | 单只不超过 5%、回撤超 10% 降仓 |

量化的优势在于：**纪律性**（不受情绪干扰）、**可检验性**（能用历史数据回测）、**可扩展性**（一套程序能同时盯上千只股票）。它的局限同样明确：**历史不代表未来**，模型会失效，市场会变化，再精妙的策略也可能在结构性转变面前崩溃。

!!! note "量化 ≠ 高频，也 ≠ 稳赚"
    很多人一听量化就想到「高频交易」。事实上高频只是量化的一个极端分支，需要昂贵的硬件与通道。本书面向的是**中低频**策略（日频、周频为主），普通人用一台笔记本就能研究。另外请牢记：量化**不等于稳赚**，它只是把投资变得**可分析、可改进**。

## 1.4 本书地图：一条完整的量化流水线

把一个想法变成可信赖的策略，要走完下面这条路。本书的六个部分正是按这条路组织的：

```text
   原始数据          可用信号           策略与组合          可信结论
┌───────────┐   ┌────────────┐   ┌──────────────┐   ┌──────────────┐
│ 行情/财务 │ → │ 技术指标   │ → │ 选股/择时    │ → │ 回测/绩效    │
│ 另类数据  │   │ 因子       │   │ 组合/配置    │   │ 风控/实盘    │
└───────────┘   └────────────┘   └──────────────┘   └──────────────┘
   第二部分         第三部分          第四部分           第五、六部分
```

- **第一部分（第1–4章）入门与基础**：Python、NumPy/Pandas、收益率与统计——工具箱。
- **第二部分（第5–7章）数据**：把真实/内置数据拿到手并洗干净——一切的起点。
- **第三部分（第8–10章）信号与因子**：从价格里提取有预测力的信号，并严谨地检验它。
- **第四部分（第11–14章）策略**：把信号组装成完整策略与投资组合。
- **第五部分（第15–17章）回测**：诚实地检验策略，识破各种「看起来很美」的假象。
- **第六部分（第18–20章）风险与实盘**：管理风险，并把策略推向真实交易。

## 1.5 中国 A 股的制度特征

在 A 股做量化，有些「常识」和美股完全不同。无视它们，回测就会系统性地高估收益。下面是最关键的几条：

| 制度 | 内容 | 对策略的影响 |
|---|---|---|
| **T+1 交易** | 当天买入的股票，次日才能卖出 | 不能做日内回转；信号要用 `shift(1)` 错开一天，避免「当天信号当天成交」的前视偏差 |
| **涨跌停限制** | 主板涨跌幅一般限制在 ±10%（创业板/科创板 ±20%） | 涨停时你想买也买不进、跌停时想卖也卖不出，回测里假设「按收盘价成交」会失真 |
| **停牌** | 公司可因重大事项停牌，期间无法交易 | 数据里会出现长段缺失，处理不当会引入幸存者偏差 |
| **做空受限** | 融券标的有限、成本高、可能无券可融 | 多数 A 股策略以**纯多头**为主，「多空对冲」在 A 股落地难度大 |

!!! warning "T+1 与前视偏差"
    A 股最常见的回测错误，就是用「今天收盘后才知道的信号」去「今天收盘成交」。今天收盘的数据你今天收盘时其实还来不及用——更何况 T+1 下今天买的明天才能卖。正确做法是让信号至少滞后一天：`持仓 = 信号.shift(1)`。这一条会在第15章反复强调。

## 1.7 第一段代码：加载行情、算收益、画净值

下面这段代码不需要联网，只用内置数据。它做了三件事：加载一只股票的收盘价、计算累计净值、画出来。完整可运行版本见配套 notebook。

```python
import matplotlib.pyplot as plt

from qfin import load_close, set_chinese_font

set_chinese_font()                       # 统一中文绘图样式

close = load_close()                     # 收盘价宽表：行=日期，列=股票
one = close["STK001"]                    # 取一只股票
nav = one / one.iloc[0]                  # 归一化为「净值」（起点=1）

nav.plot(title="STK001 净值曲线（起点归一为 1）")
plt.xlabel("日期")
plt.ylabel("净值")
plt.show()

print(f"区间累计收益：{nav.iloc[-1] - 1:.1%}")
```

!!! example "例 1.1　从价格到收益"
    若某股票期初价 10 元、期末价 13 元，则区间累计收益为 $13/10 - 1 = 30\%$。把价格序列整体除以期初价，就得到**净值曲线**（起点为 1），它把不同价位的标的放到同一把尺子上比较——这是全书最基础、也最常用的一步。

正式的收益率定义、对数收益、年化与风险度量，将在第4章系统展开。现在你只需要确认：**环境能跑、数据能加载、图能画出来**。

## 1.8 本章小结

### 制度理解

**习题1.3（T+1 的影响）** 解释为什么在 A 股回测中「持仓 = 信号」会高估收益，而「持仓 = 信号.shift(1)」更合理。

??? tip "参考思路"
    信号通常基于收盘价计算，意味着你要到收盘才知道该不该买；而 T+1 下今天买的明天才能卖。`shift(1)` 让信号滞后一天成交，避免使用「未来才知道」的信息。
